Начало работы

Лучший способ начать работу со стеком Reticulum Network зависит от ваших целей. Для полного описания и примеров см. раздел Getting Started Fast в Руководстве Reticulum.

Экспериментальное программное обеспечение

Внимание! Все основные функции протокола реализованы и работают, но, вероятно, будут добавления по мере изучения реального использования. API и формат передачи данных можно считать завершёнными и стабильными, однако они могут измениться, если это будет абсолютно необходимо. Используйте Reticulum, только если вы согласны с этим и понимаете последствия.

Установка

Самый простой способ установить Reticulum и сопутствующие утилиты — через pip:

pip install rns

После этого можно запускать любую программу, использующую Reticulum, или запустить Reticulum как системный сервис с утилитой rnsd.

Если pip недоступен, сначала установите пакеты python3 и python3-pip для вашей ОС.

Конфигурация

При первом запуске Reticulum создаст файл конфигурации по умолчанию, обеспечивая базовую связность с другими узлами Reticulum, доступными локально. Если среди этих узлов есть транспортные экземпляры, они могут подключить вас к более широким сетям. Файл конфигурации содержит несколько примеров и ссылки для создания более сложных настроек.

Поддерживаемые интерфейсы

Подробные примеры расширения связи через различные среды — пакетное радио, LoRa, последовательные порты, быстрые IP‑соединения и Интернет через UDP и TCP — см. раздел Supported Interfaces в Руководстве Reticulum.

Включённые утилиты

Reticulum поставляется с набором полезных утилит для управления сетями, просмотра статуса и информации, а также других задач. Подробнее о них в разделе Included Utility Programs в Руководстве Reticulum.

  • Системный демон rnsd для постоянного сервиса Reticulum
  • Утилита статуса интерфейсов rnstatus, показывающая информацию об интерфейсах
  • Инструмент управления путями rnpath для просмотра и изменения таблиц маршрутизации
  • Диагностическая утилита rnprobe для проверки связности с узлами
  • Простая программа передачи файлов rncp для копирования файлов на удалённые системы
  • Утилита удалённого выполнения команд rnx, позволяющая запускать команды и получать вывод с удалённых систем

Все инструменты, включая rnx и rncp, работают надёжно даже по очень низкоскоростным каналам, таким как LoRa или пакетное радио.

Программы, использующие Reticulum

Чтобы быстро понять возможности Reticulum, ознакомьтесь со следующими ресурсами.

  • Для автономной, зашифрованной и устойчивой mesh‑сети смотрите Nomad Network
  • Приложения для Android, Linux и macOS — Sideband, имеющие графический интерфейс и ориентированные на простоту использования.
  • LXMF — распределённый, tolerant‑to‑delay и disruption‑tolerant протокол передачи сообщений, построенный на Reticulum

Зависимости

Установка базового пакета rns требует перечисленных ниже зависимостей. На большинстве систем и дистрибутивов они доступны в виде пакетов, и при установке rns через pip они будут автоматически загружены и установлены.

  • PyCA/cryptography
  • netifaces
  • pyserial

Вариант Pure‑Python

На необычных системах, где один или несколько из перечисленных модулей нельзя установить, можно использовать пакет rnspure, который не требует внешних зависимостей при установке. Содержимое пакетов rns и rnspure идентично; единственное различие — rnspure не содержит зависимостей.

Reticulum загружает внешние зависимости только если они необходимы и доступны. Например, если pyserial недоступен, интерфейсы на основе последовательного порта просто не будут работать, тогда как остальные модули загрузятся при необходимости.

Обратите внимание! Если вы используете rnspure на системах, не поддерживающих PyCA/cryptography, внимательно изучите раздел Криптографические примитивы.

Производительность

Reticulum нацелен на очень широкий диапазон практической производительности, но отдаёт приоритет функциональности и производительности над низкоскоростными средами. Цель — обеспечить динамический диапазон от 250 бит в секунду до 1 гигабита в секунду на обычном оборудовании.

В текущий момент практический диапазон составляет примерно от 150 бит в секунду до 40 мегабит в секунду, при этом более быстрые физические среды не полностью загружены. Повышение производительности планируется в будущих версиях, но не является приоритетом, пока не будет окончательно определён формат передачи и API.
